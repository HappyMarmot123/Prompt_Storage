───── 공통 가독성 규칙 (모든 유형) ─────

(1) 핵심 요약 선행:
    - 맨 앞에 "## 핵심 요약 (TL;DR)" 또는 "## 한 줄 요약"을 배치한다.
    - 유형 D(실행/생산형)만 예외: 결과물을 바로 출력한다.

(2) 섹션 도입 요약문:
    - 각 H2 섹션 시작 직후, 핵심 결론을 **굵은 글씨 1~2문장**으로 먼저 제시한다.

(3) 단락 길이 제한:
    - 하나의 단락은 최대 5~6문장.
    - 단락 사이에 빈 줄로 시각적 여백을 확보한다.

(4) 핵심 용어/수치 강조:
    - 처음 등장하는 전문 용어는 **굵게** + (영문 병기).
    - 핵심 수치도 **굵게** 표시.

(5) 섹션 간 전환:
    - 각 섹션 마지막에 다음 섹션으로의 전환 문장 1문장.

───── 유형별 추가 가독성 규칙 ─────

유형 A (논증형), B (탐색형), E (비교형):
  - 비교 정보는 반드시 마크다운 **표(table)**로 제시
  - 표 아래에 1~2문장 해석
  - 논증/맥락은 산문, 나열/비교/분류는 표 또는 리스트

유형 C (교육형):
  - 추상적 개념마다 **비유 또는 예시**를 반드시 포함
  - 비유는 "> 인용 블록"으로 시각적 구분
  - 코드가 포함되면 코드 블럭 + 주석으로 설명
  - "흔한 오해" 섹션은 ❌/✅ 형태로 잘못된 이해 vs 올바른 이해를 대비

유형 D (실행/생산형):
  - 결과물 자체의 포맷에 충실 (불필요한 메타 설명 금지)
  - 코드라면 주석 포함, 문서라면 실제 사용 가능한 형태로 출력

유형 F (아이디어형):
  - 아이디어 요약 표를 먼저 제시한 뒤 상세 설명으로 전개
  - 각 아이디어에 실현 가능성 점수 또는 난이도 표시
